# Реализация фильтрации статей по тегам

## Обзор

Система фильтрации статей по тегам реализована с использованием React Context API и Docusaurus swizzling. Она позволяет пользователям:

1. Выбирать теги в компоненте фильтра
2. Видеть, как список статей в сайдбаре динамически обновляется
3. Фильтровать статьи по одному или нескольким тегам одновременно

## Архитектура

### Компоненты

#### 1. **TagFilterContext** (`src/contexts/TagFilterContext.tsx`)
- Управляет состоянием выбранных тегов
- Предоставляет методы для добавления/удаления тегов
- Доступен через хук `useTagFilter()`

#### 2. **PaperTagFilter** (`src/components/PaperTagFilter/index.tsx`)
- Отображает все доступные теги, организованные по категориям
- Позволяет выбирать/деселектировать теги
- Использует `useTagFilter()` для управления состоянием

#### 3. **PaperSection** (`src/components/PaperSection/index.tsx`)
- Обертка для раздела статей
- Предоставляет `TagFilterProvider` для всех дочерних компонентов

#### 4. **Root** (`src/theme/Root.tsx`)
- Корневой компонент приложения
- Обеспечивает доступ к `TagFilterContext` для всех компонентов

#### 5. **DocSidebar (Desktop/Mobile)** (`src/theme/DocSidebar/`)
- Swizzled компоненты Docusaurus
- Фильтруют элементы сайдбара на основе выбранных тегов
- Скрывают статьи, которые не содержат выбранные теги

## Как это работает

### Поток данных

```
User selects tag
    ↓
PaperTagFilter calls toggleTag()
    ↓
TagFilterContext updates selectedTags
    ↓
DocSidebar (Desktop/Mobile) receives updated selectedTags
    ↓
filterSidebarItems() filters papers
    ↓
Sidebar re-renders with filtered papers
```

### Фильтрация

Функция `filterSidebarItems()` рекурсивно проходит по элементам сайдбара:

1. Для категорий: фильтрует элементы внутри категории, скрывает пустые категории
2. Для документов (статей): проверяет, содержит ли статья хотя бы один из выбранных тегов
3. Для других элементов: показывает как есть

### Извлечение тегов

Функция `extractTagsFromDoc()` пытается получить теги из различных источников:
- `doc.tags`
- `doc.metadata.tags`
- `doc.frontMatter.tags`

## Использование

### Для пользователей

1. Перейти на страницу "Статьи" (`/docs/papers`)
2. Выбрать один или несколько тегов в фильтре
3. Сайдбар автоматически обновится, показывая только релевантные статьи
4. Нажать "Очистить" для сброса фильтра

### Для разработчиков

#### Использование контекста в компоненте

```tsx
import { useTagFilter } from '@site/src/contexts/TagFilterContext';

function MyComponent() {
  const { selectedTags, toggleTag, clearTags } = useTagFilter();
  
  // Ваш код здесь
}
```

#### Добавление новых тегов

1. Обновить `TAG_KEYWORDS` в `scripts/collect_papers_extended.py`
2. Обновить `tagCategories` в `src/components/PaperTagFilter/index.tsx`
3. Запустить `regenerate_paper_tags.py` для переиндексирования существующих статей

## Ограничения и будущие улучшения

### Текущие ограничения

1. **Доступ к тегам**: Docusaurus не предоставляет прямого доступа к метаданным документов через API. Теги извлекаются из объекта документа, который может быть недоступен в некоторых случаях.

2. **Производительность**: При большом количестве статей фильтрация может быть медленной. Можно оптимизировать с помощью мемоизации.

3. **Сохранение состояния**: Выбранные теги не сохраняются при перезагрузке страницы. Можно добавить сохранение в localStorage.

### Будущие улучшения

1. **Сохранение состояния в URL**: Добавить параметры в URL для сохранения выбранных тегов
2. **Сохранение в localStorage**: Запомнить выбранные теги между сеансами
3. **Счетчики**: Показать количество статей для каждого тега
4. **Поиск по тегам**: Добавить поле поиска для быстрого поиска тегов
5. **Экспорт**: Позволить пользователям экспортировать отфильтрованные статьи

## Отладка

### Проверка, работает ли фильтрация

1. Откройте DevTools (F12)
2. Перейдите на вкладку "Console"
3. Выполните:
   ```js
   // Проверить, доступен ли контекст
   console.log('TagFilterContext is available');
   ```

### Проверка тегов в документе

1. Откройте любую статью
2. В DevTools выполните:
   ```js
   // Найти элемент с метаданными
   console.log(document.querySelector('[data-doc-id]'));
   ```
