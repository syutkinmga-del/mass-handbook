# Гайд по разработке системы избежания столкновений

Практическое руководство по проектированию и реализации системы автоматического избежания столкновений для морских автономных судов (MASS).

## Введение

Система избежания столкновений является критическим компонентом любого автономного судна. Она должна соответствовать международным правилам (COLREGs) и обеспечивать безопасность в различных морских условиях.

## Архитектура системы

Типичная система избежания столкновений состоит из следующих компонентов:

### 1. Модуль восприятия (Perception Module)

**Задачи:**
- Обнаружение других судов и препятствий
- Оценка расстояния и азимута
- Отслеживание движущихся объектов
- Предсказание траектории других судов

**Входные данные:**
- Данные радара (расстояние, азимут, скорость)
- Данные от камер (визуальное распознавание)
- AIS (Automatic Identification System) - информация о других судах
- GPS/GNSS - позиция собственного судна
- Компас и гироскоп - курс и угловая скорость

**Выходные данные:**
- Список обнаруженных объектов с параметрами (расстояние, азимут, скорость, курс)
- Оценка неопределенности для каждого объекта

### 2. Модуль ситуационной осведомленности (Situational Awareness Module)

**Задачи:**
- Определение типа встречи судов (head-on, crossing, overtaking)
- Оценка риска столкновения (CPA - Closest Point of Approach, TCPA - Time to CPA)
- Определение применимых правил COLREGs
- Прогнозирование действий других судов

**Ключевые метрики:**
- **CPA (Closest Point of Approach):** Минимальное расстояние между судами при текущих курсах и скоростях
- **TCPA (Time to CPA):** Время до наименьшего сближения
- **Risk Level:** Оценка риска столкновения (низкий, средний, высокий)

**Пример расчета:**

```
Собственное судно: позиция (x₀, y₀), скорость (vₓ, vᵧ)
Другое судно: позиция (x₁, y₁), скорость (vₓ, vᵧ)

Относительная позиция: Δx = x₁ - x₀, Δy = y₁ - y₀
Относительная скорость: Δvₓ = vₓ - vₓ, Δvᵧ = vᵧ - vᵧ

CPA = |Δx × Δvᵧ - Δy × Δvₓ| / √(Δvₓ² + Δvᵧ²)
TCPA = -(Δx × Δvₓ + Δy × Δvᵧ) / (Δvₓ² + Δvᵧ²)
```

### 3. Модуль принятия решений (Decision Making Module)

**Задачи:**
- Определение необходимости маневра
- Выбор типа маневра (изменение курса, изменение скорости, оба)
- Выбор параметров маневра (новый курс, новая скорость)
- Обеспечение соответствия COLREGs

**Алгоритм принятия решений:**

1. Если CPA > безопасное расстояние и TCPA > времени реакции → Маневр не требуется
2. Если CPA ≤ безопасное расстояние → Требуется маневр
3. Определить тип встречи (head-on, crossing, overtaking)
4. Выбрать маневр согласно COLREGs:
   - **Head-on:** Оба судна должны изменить курс вправо
   - **Crossing:** Судно, имеющее другое судно справа, должно уступить дорогу
   - **Overtaking:** Судно, совершающее обгон, должно уступить дорогу
5. Оптимизировать параметры маневра (минимизировать отклонение от маршрута)

### 4. Модуль управления (Control Module)

**Задачи:**
- Реализация выбранного маневра
- Управление курсом и скоростью
- Мониторинг выполнения маневра
- Адаптация при изменении ситуации

## Выбор алгоритма планирования пути

### Dynamic Window Approach (DWA)

**Преимущества:**
- Реактивный (низкая задержка)
- Учитывает динамические ограничения судна
- Хорошо работает в динамичной среде

**Недостатки:**
- Может застревать в локальных минимумах
- Требует хорошей оценки препятствий

**Применение:** Локальное избежание столкновений в реальном времени

### Model Predictive Control (MPC)

**Преимущества:**
- Оптимальное управление
- Явный учет ограничений
- Предсказательный подход

**Недостатки:**
- Высокие вычислительные требования
- Требует точной математической модели

**Применение:** Маневры в узких проливах, причаливание

### Deep Reinforcement Learning (DRL)

**Преимущества:**
- Может обучиться оптимальной политике
- Адаптируется к новым условиям
- Может обрабатывать сложные сценарии

**Недостатки:**
- Требует больших объемов данных для обучения
- Сложность интерпретации решений
- Требует валидации на соответствие COLREGs

**Применение:** Обучение оптимальным маневрам в симуляции

## Реализация

### Псевдокод основного цикла

```python
def collision_avoidance_loop():
    while True:
        # 1. Получить данные от датчиков
        perception_data = get_sensor_data()
        
        # 2. Обновить модель окружающей среды
        detected_objects = perception_module.process(perception_data)
        
        # 3. Оценить ситуацию
        situation = situational_awareness.analyze(detected_objects, own_position, own_velocity)
        
        # 4. Проверить риск столкновения
        if situation.risk_level > RISK_THRESHOLD:
            # 5. Принять решение о маневре
            maneuver = decision_making.decide(situation)
            
            # 6. Проверить соответствие COLREGs
            if is_colregs_compliant(maneuver):
                # 7. Выполнить маневр
                control_module.execute(maneuver)
        
        # Задержка цикла (обычно 1-10 Гц)
        sleep(0.1)
```

## Тестирование и валидация

### Сценарии тестирования

1. **Head-on встреча:** Два судна движутся навстречу друг другу
2. **Crossing встреча:** Два судна пересекают курсы друг друга
3. **Overtaking сценарий:** Одно судно обгоняет другое
4. **Множественные препятствия:** Несколько судов в поле зрения
5. **Динамические препятствия:** Препятствия, появляющиеся неожиданно
6. **Отказ датчиков:** Потеря данных от одного или нескольких датчиков

### Метрики оценки

- **Успешность избежания столкновений:** % сценариев, где столкновение успешно избежано
- **Соответствие COLREGs:** % решений, соответствующих правилам
- **Время реакции:** Время от обнаружения препятствия до начала маневра
- **Отклонение от маршрута:** Дополнительное расстояние, пройденное из-за маневров
- **Комфортность маневра:** Максимальное ускорение и угловая скорость

## Лучшие практики

1. **Безопасность в первую очередь:** Всегда предпочитайте безопасность оптимальности маршрута
2. **Консервативные пороги:** Используйте консервативные пороги для срабатывания маневров
3. **Резервирование:** Имейте резервные системы на случай отказа основной системы
4. **Мониторинг:** Постоянно мониторьте состояние системы и логируйте все решения
5. **Обучение:** Регулярно анализируйте логи и улучшайте алгоритмы
6. **Валидация:** Перед развертыванием проведите тщательное тестирование в симуляции и в контролируемых условиях
