# Гайд по разработке системы планирования пути

Практическое руководство по проектированию и реализации системы планирования пути для морских автономных судов (MASS).

## Введение

Планирование пути — это процесс определения оптимальной траектории движения судна от начальной точки к целевой, с учетом препятствий, ограничений и целей. Эффективное планирование пути критически важно для безопасности, экономичности и соответствия морским правилам.

## Уровни планирования

### Глобальное планирование (Strategic Planning)

**Масштаб:** Часы-дни, расстояния в сотни морских миль

**Задачи:**
- Выбор оптимального маршрута между портами
- Учет морских коридоров и ограничений
- Прогнозирование погоды и волнения
- Минимизация расхода топлива

**Методы:**
- Алгоритмы на основе графов (A*, Dijkstra)
- Вероятностные методы (RRT, PRM)
- Оптимизация на основе метеоданных

### Тактическое планирование (Tactical Planning)

**Масштаб:** Часы, расстояния в десятки морских миль

**Задачи:**
- Адаптация маршрута при изменении условий
- Обход зон, запрещенных для судоходства
- Планирование встреч с другими судами
- Оптимизация скорости

**Методы:**
- Динамическое переплани рование
- Модель предиктивного управления (MPC)
- Оптимизация на основе текущих условий

### Локальное планирование (Reactive Planning)

**Масштаб:** Минуты, расстояния в несколько морских миль

**Задачи:**
- Избежание неожиданных препятствий
- Реактивное управление курсом и скоростью
- Обеспечение безопасности в реальном времени

**Методы:**
- Dynamic Window Approach (DWA)
- Potential Field Methods
- Быстрые алгоритмы оптимизации

## Выбор алгоритма

### A* (A-Star)

**Применение:** Глобальное планирование на дискретной сетке

**Преимущества:**
- Гарантированно находит оптимальный путь
- Эффективен при наличии хорошей эвристики
- Хорошо изучен и документирован

**Недостатки:**
- Требует дискретизации пространства
- Может быть медленным на больших сетках
- Не учитывает динамические препятствия

**Пример использования:**

```python
import heapq

def a_star(start, goal, grid, heuristic):
    """A* алгоритм для планирования пути"""
    open_set = [(0, start)]
    came_from = {}
    g_score = {start: 0}
    f_score = {start: heuristic(start, goal)}
    
    while open_set:
        current = heapq.heappop(open_set)[1]
        
        if current == goal:
            return reconstruct_path(came_from, current)
        
        for neighbor in get_neighbors(current, grid):
            tentative_g = g_score[current] + distance(current, neighbor)
            
            if neighbor not in g_score or tentative_g < g_score[neighbor]:
                came_from[neighbor] = current
                g_score[neighbor] = tentative_g
                f_score[neighbor] = tentative_g + heuristic(neighbor, goal)
                heapq.heappush(open_set, (f_score[neighbor], neighbor))
    
    return None  # Путь не найден
```

### Rapidly-exploring Random Trees (RRT)

**Применение:** Глобальное планирование в непрерывном пространстве

**Преимущества:**
- Работает в непрерывном пространстве
- Хорошо работает с высокой размерностью
- Может найти путь в сложных окружениях

**Недостатки:**
- Не гарантирует оптимальность
- Может быть медленным
- Требует хорошей настройки параметров

### Dynamic Window Approach (DWA)

**Применение:** Локальное планирование в реальном времени

**Преимущества:**
- Реактивный (низкая задержка)
- Учитывает динамические ограничения
- Хорошо работает в динамичной среде

**Недостатки:**
- Может застревать в локальных минимумах
- Требует хорошей оценки препятствий

## Интеграция с системой избежания столкновений

Система планирования пути должна тесно интегрироваться с системой избежания столкновений:

1. **Глобальный маршрут:** Система планирования пути определяет желаемый маршрут
2. **Локальная адаптация:** Система избежания столкновений адаптирует маршрут при обнаружении препятствий
3. **Обратная связь:** Информация о выполненных маневрах передается обратно в систему планирования
4. **Переплани рование:** При значительных отклонениях система пересчитывает маршрут

## Оптимизация маршрута

### Критерии оптимизации

1. **Расстояние:** Минимизация пройденного расстояния
2. **Время:** Минимизация времени в пути
3. **Расход топлива:** Минимизация расхода топлива (зависит от скорости и условий)
4. **Безопасность:** Максимизация расстояния от препятствий
5. **Комфортность:** Минимизация ускорений и угловых скоростей

### Функция стоимости

```
Cost = w₁ × Distance + w₂ × Time + w₃ × FuelConsumption 
       - w₄ × Safety + w₅ × Comfort

где w₁, w₂, w₃, w₄, w₅ — весовые коэффициенты
```

## Тестирование и валидация

### Сценарии тестирования

1. **Открытое море:** Планирование маршрута без препятствий
2. **Прибрежная зона:** Планирование маршрута с учетом береговой линии
3. **Морские коридоры:** Планирование маршрута через установленные коридоры
4. **Динамические препятствия:** Адаптация маршрута при появлении других судов
5. **Экстремальные условия:** Планирование маршрута в штормовых условиях

### Метрики оценки

- **Оптимальность:** Отношение длины найденного пути к оптимальному
- **Время вычисления:** Время, необходимое для расчета маршрута
- **Успешность:** % сценариев, где маршрут успешно найден
- **Гладкость:** Отсутствие резких изменений курса
- **Соответствие правилам:** % маршрутов, соответствующих морским правилам

## Лучшие практики

1. **Иерархический подход:** Используйте разные алгоритмы для разных уровней планирования
2. **Переплани рование:** Регулярно пересчитывайте маршрут при изменении условий
3. **Безопасные запасы:** Используйте безопасные запасы расстояния от препятствий
4. **Валидация:** Проверяйте найденный маршрут на соответствие правилам
5. **Мониторинг:** Отслеживайте отклонения от плана и адаптируйте маршрут
6. **Документирование:** Логируйте все решения по планированию маршрута
